Low-Level Settings

This page describes preferences that generally require special edits to make. For example, you may need to use the command-line “defaults” program, or edit XML directly, instead of interacting with a window in MacTerm.

Overview

MacTerm preferences are hierarchical. Every default value is in “net.macterm.MacTerm.plist” (in the Preferences folder of the user Library folder), even if it appears to be a Default collection in the user interface.

Collections themselves are separate property list files, of the form “net.macterm.MacTerm.[type].[number].plist”. The names of these domains are recorded in the primary defaults file.

Each collection file contains either a “name” key of type Data, or a “name-string” key of type String, or both. The String version is less precise, but is useful for easy viewing and debugging of names. The Data version is preferred and is an external representation of Unicode. These names determine how MacTerm displays the collection in any user interface, such as a menu or a list in the Preferences window.

MacTerm settings are designed so that any preferences key might one day fit into any collection. However, currently, only related settings are actually expected; for instance, nothing will check for colors in a Session collection.

Macro Collections

macro-NN-contents-string, where NN is 01 to 12 string
Text, whose meaning depends on the action: something to send (interpreted or verbatim), a URL to open, or a command line to run (without interpretation).
macro-NN-key, where NN is 01 to 12 string: a key name
Base key that activates this macro, when combined with the modifier keys.
macro-NN-modifiers, where NN is 01 to 12 array of string: zero or more modifier keys
Modifier keys that must be held down before the base key is interpreted as a macro request.
macro-NN-name-string, where NN is 01 to 12 string
Name to use in menus and other interface elements, when representing this macro.
macro-order array of string: values from macro-01 to macro-12
Order in which macros should be displayed in user interface elements.

Special Values

Sometimes, several preference keys accept similar values, that can have a wide range of valid tokens.

A key name can describe many keys on the keyboard. Characters that can unambiguously describe a key (such as capital letters) are all valid. In addition, the string value could be return, enter, escape, clear, forward-delete, backward-delete, home, end, page-up, page-down, left-arrow, right-arrow, up-arrow, down-arrow, or f1 through f16.

A modifier key is described by a string value equal to shift, command, control, or option.
