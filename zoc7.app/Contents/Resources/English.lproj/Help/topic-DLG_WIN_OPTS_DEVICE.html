Connection Type Options

This window is used to select and configure the available connection methods. It can be opened via Options menu or by clicking on the leftmost button in the status bar.  

  

 

Connection Type

This list defines the standard operation parameters for all connection methods. It also selects the default connection method for this session profile. 

However, the selection and settings made here may later be ignored if the connection, which the session profile was assigned to, overrides them. 

For example, if the session profile is assigned to a Host Directory Entry, the entry itself can be set to use a different connection type. The entry can also choose to use or to override the session profile's options for that particular type. 

In other situations, e.g. if connections are made from the command line or if the profile is activated throgh REXX scripts, the settings will be used as they are defined here. 

  

Help for the individual methods is available in the Connection Type section of this help text and you will find details for the specific types in the list below:

  • Secure Shell
  • Telnet
  • Serial/Modem
  • Windows-Modems
  • ISDN
  • Rlogin
  • Named Pipe

 

Additionally you can set an option to release the connection handler when the ZOC window is minimized. However, this is only necessary with Serial/Modem, because this is the only method that exclusively locks a system resource (the com port).
