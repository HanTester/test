Global SSH authentication Files

Global authentication files are useful if you connect to a variety of hosts, where you can log in with the same public/private (identity) key file. 

These files are used for a login, when you make a connection via Quick Connection or via Host Directory and if the option to log in via global authentication files is checked. They are also always used, if you have set the option to use an identity file in the Global SSH authentication dialog. 

The Global Authentication Files dialog lets you specify three different types of public/private key files, which will be used depending on the SSH version (RSA for SSH-V1 and SSH-V2, DSA for SSH-V2).
